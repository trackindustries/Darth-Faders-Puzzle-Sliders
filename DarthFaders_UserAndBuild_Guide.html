Darth Faders Puzzle/Trigger

A 3-slider motorized puzzle using a Feather RP2040, motorized slide potentiometers, a bi-color 24-bar bargraph, and color-coded step switches which trigger a 3.5v signal to a dedicated pin to initiate or trigger another device. We used this as a way to unlock a secret door in our home. You could also use this as a puzzle in an escape room. Based on the Adafruit Darth Faders project by John Park. You can use that project for a guide to set up. We have not modified their 3D files, only added a top crown unit to hold the stepper switches and number plate for backpack. You can download the entire 3D print file bundle in our GitHub repository mentioned later in this project

CODE: C++ CONTROLER: Feather RP2040 LOE: High; 8+ Hrs 3D PRINTING: Yes SKILL LEVEL: Moderate

ℹ️
MOD/BRANCH IDEAS: The selected motorized sliders have a touch pin. This means if you utilized a capacitive slider knob, many of the functions which use the stepper switches could be eliminated simply by touching the slider.
🎛️

Introduction

Three motorized sliders each represent a value from 1–24. Players must move each slider to the correct position and confirm it using the matching colored step switch. When all three values match the stored code, the puzzle triggers a success output on a designated GPIO pin.

An admin (momentary) button enables setting a new code, revealing the current code, and factory resetting — all without modifying firmware.

ℹ️
The solve code is stored in flash via the EEPROM library (built into the Arduino-Pico core) and survives power cycles and firmware re-uploads (unless you erase all flash).

ADC Calibration

✅
No manual calibration required. The firmware automatically calibrates all three sliders on every boot — no measuring, no code changes, no re-uploading.

On every power-up, calibrateSliders() runs automatically inside puzzleInit() before the puzzle becomes active. It physically sweeps all three sliders through their full range, reads the real ADC values from your specific hardware, and sets the calibration range in memory. The puzzle then starts with accurate, hardware-matched calibration every single time.

What you'll see on boot

  1. All three sliders drive to the physical bottom simultaneously.
  2. The bargraph fills red from top to bottom — confirming the low readings have been taken.
  3. All three sliders drive to the physical top simultaneously.
  4. The bargraph fills green from bottom to top — confirming the high readings have been taken.
  5. All three sliders return to the bottom. Bargraph clears. Puzzle enters IDLE and is ready.

How it works

The calibration sweep takes the lowest raw ADC reading across all three sliders at the bottom position and the highest reading at the top, then trims a ADC_CAL_MARGIN (50 counts) off each end as a safety buffer. The resulting values are stored in the runtime variables adcMin and adcMax, which readSlider() uses to map raw ADC values to the 1–24 step range.

If the calibration produces a degenerate result — for example if a motor isn't connected — the firmware falls back to the ADC_MIN_DEFAULT / ADC_MAX_DEFAULT constants in config.h and logs a warning to Serial.

Verifying calibration via Serial Monitor

Open Serial Monitor at 115200 baud during boot to see the raw readings for each slider and the final computed range:

Setting the Solve Code

ℹ️
Prerequisite: No step switches may be in solve mode (all LEDs off) when you begin the hold.
  1. Hold the momentary button for 5 seconds → enters SET mode. All three step switch LEDs blink fast.
  2. Press any step switch (e.g. Red) → its LED changes to slow blink, indicating you are now setting that color's value.
  3. Move that slider up or down → the bargraph fills in the corresponding color, showing the current value (1–24).
  4. Press the same step switch again → value is locked. LED goes solid. Bargraph shows a single LED at the locked position.
  5. Repeat steps 2–4 for the remaining two colors. When each is set its LED is solid; unset colors continue to blink fast.
  6. If two colors share the same value, the bargraph LED at that position will alternate between both colors every 500ms.
  7. Once all three LEDs are solid, press the momentary button once → code is saved to EEPROM flash → celebration sequence plays → puzzle returns to IDLE.
🧩

Solving the Puzzle

  1. From IDLE, press any step switch → its LED blinks slow (that color is now in solve mode).
  2. Move that slider → the bargraph fills in that color, showing position 1–24 in real time.
  3. Press the same switch again → value locked. LED goes solid.
  4. Repeat for the other colors in any order. Additional colors can be placed in solve mode at any time by pressing their switch.
  5. When all three LEDs are solid, the code is evaluated automatically.

✅ Correct Code

All LEDs flash very fast for 4 seconds → sliders perform victory dance (Center → Top → Bottom → Top → Bottom) → D24 (GPIO24) goes HIGH for 3 seconds → puzzle returns to IDLE.

❌ Wrong Code

All LEDs go off → bargraph clears → all sliders return to the bottom (position 1) → puzzle returns to IDLE.

👁️

Reveal Mode

Allows the puzzle operator to view the currently stored solve code without resetting it.

  1. Press all three step switches (any order) so all three LEDs are blinking slowly.
  2. Hold the momentary button for 5 seconds → enters REVEAL mode.
  3. All three sliders motor to their stored solve positions. The bargraph displays all three values in their respective colors.
  4. Press the momentary button once → all LEDs turn off, sliders return to bottom, bargraph clears, puzzle returns to IDLE.
🔄

Factory Reset

⚠️
The stored solve code will be overwritten with 7 / 7 / 7.
  1. Ensure the puzzle is in IDLE state with all step switch LEDs off (no switches in solve mode).
  2. Hold the momentary button for 10 seconds.
  3. After 5 seconds you will be in SET mode — the step switch LEDs will start to flash rapidly. Continue holding the button down for another 5 seconds until the bargraph illuminates.
  4. A spectrum bargraph sweep plays (top to bottom, then bottom to top) → solve code resets to 7/7/7 in EEPROM → puzzle returns to IDLE.
